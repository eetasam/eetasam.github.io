OVERVIEW
Case Study: Whoosh Mail is a webapp used to manage, send, and track email campaigns.
AUDIENCE
This app is built for small businesses owners and employees across the US.
Process Deck
PROBLEM
A Two-Flow Web App
Whoosh Mail is looking to develop an email marketing campaign web app with two distinct flows for Admin and Basic users.
INITIAL RESEARCH
After researching competitors and determining the target market, I created user test flows for the Admin and Basic users.
Admin UTF
View the Basic User Persona and Storyboard.
PROJECT TARGETS
My goal was to develop a web app that effectively increased usability of Whoosh Mail for basic and administrative users. I focused on the following features in my design:
  • After looking at the competitor websites, I definitely want to aim to have a very clean and intuitive interface, free from clutter.
  • On my template gallery page, I want current campaigns to be presented front and foremost, as they are good reference points for the user.
  • The gallery should be organized by template type.
  • I was most interested in the layout for the email editor page, as this is arguably the most important part of the web app.
  • I experimented with different ways to divide up the sections of the template for editing, and I decided on this bookmark-y type of layout, with little pullout sections that the user can move between.

FINAL PROTOTYPES
After reviewing my wireframes, I moved to create high-fidelity mockups. I maintained all design choices from the wireframes, but added additional elements such as edit tools on the template editor page.

LEARNINGS
Prototyping in Adobe XD
I really dug in and built my proficiency in Adobe XD. Specifically, I played with default/hover/click states to make my prototypes as life-like as possible. I also learned how to correctly and effectively create and use components in Adobe XD, so as to lessen the workload and increase efficiency in my development process.
WebApps
I really had no idea how to approach building a web app before this project. Web apps share features common in phone applications and websites, so it was definitely a learning process to figure out how to meld the two in a way that looked clean and intuitive.
